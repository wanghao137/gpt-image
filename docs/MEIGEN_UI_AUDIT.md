# MeiGen UI 对比评估记录

> 日期：2026-05-22
> 状态：仅记录，暂不进入优化开发
> 参考站点：https://www.meigen.ai/
> 对比对象：桃子AI视觉实验室（taostudioai.com / 当前本地项目）

这份记录用于沉淀竞品观察和后续优化候选项。当前不按本文直接改代码，后续需要优化时再重新校验线上状态、数据结构和移动端截图。

---

## 1. 评估结论

MeiGen 的核心优势不是单纯更漂亮，而是浏览效率和工具感更强。它把用户打开页面后的第一目标定义为“马上看图、马上筛选、马上复制 prompt”，因此首页、搜索、分类、收藏、历史、模型筛选都非常靠前。

桃子AI视觉实验室当前的优势是品牌识别、中文场景组织、模板库沉淀和编辑型首页质感。后续优化应该借鉴 MeiGen 的信息架构和交互效率，但不建议照搬它的整套视觉，否则会削弱我们已经建立起来的中文品牌和实验室气质。

---

## 2. MeiGen 的主要 UI / 交互特征

1. 首页直接进入图库，弱化品牌叙事，强化高密度视觉浏览。
2. 桌面端使用固定左侧工具栏，包含首页、搜索、生成记录、收藏、分类、最近更新和更多产品。
3. 顶部筛选入口很明显，按模型和排序快速切换，例如 All、GPT Image、Midjourney、Featured、Newest、Popular。
4. 移动端优先展示双列视觉流，首屏内容密度高，适合快速找灵感。
5. 卡片以图片为主，文字和 prompt 信息相对隐藏，视觉吸引力强，但解释性较弱。
6. 页面有 MCP Server、Skill、Figma 插件、Get Started 等入口，平台化信号明显。
7. SEO 和结构化内容较重，页面包含 prompt 文本、分类、搜索、收藏、历史等可索引入口。

---

## 3. MeiGen 值得借鉴的点

### 3.1 浏览效率

MeiGen 把案例图墙放在最前面，减少用户到达有效内容的路径。我们当前首页更像品牌型/编辑型首页，质感更强，但移动端用户需要经过 hero、数据和说明后才进入密集案例内容。

可借鉴方向：保留品牌 hero，但让案例视觉墙更早露出，尤其移动端。

### 3.2 搜索入口权重

MeiGen 把搜索作为一级入口，并提供快捷键提示。我们当前案例页有搜索和筛选，但首页顶部没有形成“搜索是核心能力”的强感知。

可借鉴方向：增加全站搜索按钮或命令面板，支持标题、分类、标签和 prompt 关键词。

### 3.3 收藏 / 历史的产品感

MeiGen 把 History 和 Favorites 放在明显位置，强化回访场景。我们已经有收藏能力，但更多是卡片级动作，不像一级产品入口。

可借鉴方向：让“我的收藏”成为顶部或移动端底部快捷入口。历史记录是否需要做，取决于后续真实使用链路。

### 3.4 筛选和排序的即时感

MeiGen 通过模型 tab 和 Featured / Newest / Popular 快速建立“可以筛、可以排序”的认知。

可借鉴方向：案例页顶部增加更明确的分段控制和排序，例如精选、最新、模板关联、可商用场景。没有真实行为数据前，不虚构热门。

### 3.5 移动端图库密度

MeiGen 移动端打开后很快进入双列图墙，适合灵感型用户快速扫图。

可借鉴方向：案例页移动端增加紧凑图墙模式，卡片文字控制在必要信息内，详情放到点击或展开后。

---

## 4. 不建议照搬的点

1. 不建议全站照搬固定侧边栏。我们的中文品牌和实验室调性更强，全站工具侧边栏会让网站变得通用、模板化。
2. 不建议把首页完全改成无解释瀑布流。MeiGen 对熟练用户高效，但对新用户指导不足。
3. 不建议堆大量模型标签。除非数据真实支持，否则不要虚标 Nanobanana、Seedance、Midjourney 等模型维度。
4. 不建议使用遮挡内容的浮层推广卡。可以借鉴持续行动入口，但不能影响图片浏览。
5. 不建议让卡片只剩图片。我们的差异化价值是中文 prompt、模板提炼和场景复用，不能弱化到只像图片站。

---

## 5. 我们当前站点的优势

1. 品牌识别更强：桃子 logo 和“桃子AI视觉实验室”更有记忆点。
2. 中文场景定位更清楚：服务小红书封面、商家海报、人像写真、信息图、模板复用等实际中文需求。
3. 模板库是差异化资产：不是单纯图库，而是“案例 + 工业模板 + 场景复用”。
4. 首页高级感更强：暗色 hero、品牌视觉、右侧动态图和精选案例更像一个有主张的视觉实验室。
5. 主题系统已经具备：浅色、深色、跟随系统是体验加分项。
6. 案例详情和复制链路更适合中文转化：目标不是只看图，而是复制、改词、生成。

---

## 6. 我们相对 MeiGen 的短板

1. 首屏浏览密度偏低，移动端尤其明显。
2. 搜索没有成为一级心智，入口权重还不够。
3. 收藏功能存在，但入口不够产品化。
4. 案例页筛选还可以更像专业素材库/工作台。
5. 移动端图库模式还可以更紧凑。
6. 更新感不够外显，最新案例、最近更新、持续维护的信号可以更明显。

---

## 7. 后续优化候选清单

### P0：优先级最高，但当前暂不执行

1. 首页首屏增加“灵感速览”露出：保留 hero，但让案例视觉墙更早出现。
2. 增加全站搜索入口：顶部按钮或搜索图标，移动端一键可达。
3. 强化案例页顶部筛选：全部、商业广告、人像写真、小红书、信息图、品牌 Logo、UI 截图、海报等。
4. 增加排序机制：先做精选、最新、模板关联、可商用场景；不虚构热门。
5. 优化移动端案例页密度：紧凑双列图墙，避免文字和图片区域混乱。

### P1：第二阶段候选

1. “我的收藏”成为一级入口。
2. 增加移动端底部快捷栏：搜索、分类、模板、收藏、回顶部。
3. 案例页桌面端增加轻量左侧筛选栏。
4. 首页增加“最近更新”模块。
5. 模板库加强与案例互链：模板展示来源案例，案例展示可套用模板。
6. 补充更完整的 WebApplication、SearchAction、ItemList 等结构化数据。

### P2：谨慎或暂缓

1. 不急着做生成器浮层，除非已有完整生成链路。
2. 不为了像 MeiGen 而整体改成浅色工具风。
3. 不上虚假的 likes / views / 热门数据。

---

## 8. 建议落地顺序

1. 先优化 `/cases`：搜索权重、筛选分段、排序、移动端紧凑图墙。
2. 再优化首页：保留品牌 hero，但提前露出更多案例。
3. 然后把收藏做成一级入口：顶部入口和移动端快捷入口。
4. 最后做模板和案例互链，把模板库做成区别于普通图库的核心资产。

---
